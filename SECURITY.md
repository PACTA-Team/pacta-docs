# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Pacta, please report it responsibly.

### How to Report

- **Email**: Send details to the PACTA Team security contact
- **Include**: Description, steps to reproduce, and potential impact

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Updates**: Progress updates every 7 days
- **Resolution**: Timeline based on severity

### Security Best Practices

When contributing to Pacta documentation or code:

1. Never commit sensitive data (API keys, credentials, tokens)
2. Use environment variables for configuration
3. Keep dependencies updated
4. Follow secure coding guidelines
